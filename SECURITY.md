## 🚨 脆弱性の報告

セキュリティの脆弱性を発見された場合は、**公開Issueを作成しないでください**。

### 報告方法

1. **メールで報告**:
   
   📧 **security@aezisai.co.jp**

2. **報告に含める情報**:
   - 脆弱性の種類
   - 影響を受けるコンポーネント/バージョン
   - 再現手順
   - 想定される影響
   - （可能であれば）修正案

### 対応タイムライン

| フェーズ | 期間 |
|---------|------|
| 初回応答 | 24時間以内 |
| 確認・評価 | 72時間以内 |
| 修正パッチ | 重大度に応じて7〜30日 |
| 公開通知 | 修正後 |

## 🛡️ セキュリティベストプラクティス

### AIシステムに関する方針

- **責任あるAI**: 公平性、透明性、説明可能性を重視
- **データプライバシー**: ユーザーデータの最小化と暗号化
- **アクセス制御**: 最小権限の原則に基づく設計

### 開発者向けガイドライン

1. **シークレット管理**: 環境変数またはシークレットマネージャーを使用
2. **依存関係**: 定期的なセキュリティアップデート
3. **コードレビュー**: セキュリティ観点を含むピアレビュー
4. **テスト**: セキュリティテストの自動化
